# 브라우저 통합 테스트 시작 프롬프트

이 프롬프트를 사용하여 브라우저를 통해 실제 시스템 통합 테스트를 시작하세요.

---

## 🚀 테스트 시작

다음 프롬프트를 복사하여 사용하세요:

```
브라우저를 열고 https://readingtree2-0.vercel.app (또는 http://localhost:3000)에 접속하세요.

Phase 1: 인증 및 온보딩 테스트를 시작합니다.

다음 문서를 참조하여 단계별로 테스트를 진행하세요:
- doc/tasks/browser-test-phase-1-auth-onboarding.md

각 시나리오를 순서대로 실행하고, 발견 사항을 기록하세요.

테스트 환경:
- 브라우저: Chrome/Edge (최신 버전)
- 개발자 도구: 활성화 (네트워크, 콘솔 모니터링)
- 테스트 계정: 카카오 또는 구글 계정 준비

시작하겠습니다.
```

---

## 📋 테스트 순서

1. **Phase 1**: 인증 및 온보딩 (`browser-test-phase-1-auth-onboarding.md`)
2. **Phase 2**: 책 관리 (`browser-test-phase-2-books.md`)
3. **Phase 3**: 기록 관리 기본 (`browser-test-phase-3-notes-basic.md`)
4. **Phase 4**: 기록 관리 고급 (`browser-test-phase-4-notes-advanced.md`)
5. **Phase 5**: 검색 (`browser-test-phase-5-search.md`)
6. **Phase 6**: 공유 (`browser-test-phase-6-share.md`)
7. **Phase 7**: 통계 및 타임라인 (`browser-test-phase-7-stats-timeline.md`)
8. **Phase 8**: 독서모임 (`browser-test-phase-8-groups.md`)
9. **Phase 9**: 프로필 관리 (`browser-test-phase-9-profile.md`)

---

## 📝 각 Phase 완료 후

각 Phase를 완료한 후:
1. 체크리스트 확인
2. 발견 사항 기록
3. 스크린샷 첨부 (필요시)
4. 다음 Phase로 진행

---

## ✅ 최종 결과 문서

모든 Phase 완료 후:
- `doc/tasks/browser-test-results-final.md` 파일 생성
- 전체 테스트 결과 통합
- 발견된 버그 및 개선 사항 정리
