# Phase 6: 공유 기능 테스트

**단계:** 6/9  
**예상 소요 시간:** 10-15분  
**필수 선행 조건:** Phase 3, 4 완료 (기록 작성 완료)

---

## 🎯 테스트 목표

1. 카드뉴스 생성이 정상 작동하는지 확인
2. 공유 링크가 정상 작동하는지 확인
3. Open Graph 메타 태그가 정상 작동하는지 확인
4. Twitter Card 메타 태그가 정상 작동하는지 확인

---

## 📋 테스트 시나리오

### 시나리오 6-1: 카드뉴스 생성

**프롬프트:**
```
기록 상세 페이지에서:

1. "공유" 버튼을 클릭합니다
2. 공유 다이얼로그가 표시되는지 확인합니다
3. 카드뉴스 생성 영역을 확인합니다
4. 템플릿을 선택합니다 (예: "minimal", "classic", "modern")
5. 카드뉴스가 생성되는지 확인합니다 (미리보기 또는 이미지 URL)
6. 개발자 도구 콘솔에서 에러가 없는지 확인합니다
7. 네트워크 탭에서 카드뉴스 생성 API 요청이 성공적으로 완료되었는지 확인합니다

확인 사항:
- 공유 버튼이 정상적으로 작동하는가?
- 공유 다이얼로그가 정상적으로 표시되는가?
- 카드뉴스 생성이 정상적으로 작동하는가?
- 템플릿 선택이 정상적으로 작동하는가?
- 생성된 카드뉴스 이미지가 정상적으로 표시되는가?
```

**예상 결과:**
- 공유 다이얼로그 표시
- 카드뉴스 생성 성공
- 카드뉴스 이미지 미리보기 표시

---

### 시나리오 6-2: 카드뉴스 다운로드

**프롬프트:**
```
카드뉴스가 생성된 후:

1. "다운로드" 버튼을 클릭합니다
2. 카드뉴스 이미지가 다운로드되는지 확인합니다
3. 다운로드된 이미지 파일을 확인합니다
4. 이미지 품질이 적절한지 확인합니다
5. 개발자 도구 콘솔에서 에러가 없는지 확인합니다

확인 사항:
- 카드뉴스 다운로드가 정상적으로 작동하는가?
- 다운로드된 이미지 파일이 정상적인가?
- 이미지 품질이 적절한가?
```

**예상 결과:**
- 카드뉴스 이미지 다운로드 성공
- 이미지 파일 정상
- 이미지 품질 적절

---

### 시나리오 6-3: 공유 링크 복사

**프롬프트:**
```
공유 다이얼로그에서:

1. "링크 복사" 버튼을 클릭합니다
2. 공유 링크가 클립보드에 복사되었는지 확인합니다 (토스트 메시지)
3. 복사된 링크를 확인합니다
4. 개발자 도구 콘솔에서 에러가 없는지 확인합니다

확인 사항:
- 링크 복사가 정상적으로 작동하는가?
- 복사된 링크가 정확한가?
- 복사 성공 피드백이 제공되는가?
```

**예상 결과:**
- 링크 복사 성공 토스트 메시지
- 복사된 링크 정확

---

### 시나리오 6-4: 공유 페이지 접근

**프롬프트:**
```
공유 링크를 사용하여:

1. 공유 링크(/share/notes/[id])를 새 탭에서 엽니다
2. 공유 페이지가 정상적으로 로드되는지 확인합니다
3. 기록 정보가 정상적으로 표시되는지 확인합니다
4. 로그인하지 않은 상태에서도 접근 가능한지 확인합니다
5. 개발자 도구 콘솔에서 에러가 없는지 확인합니다

확인 사항:
- 공유 페이지가 정상적으로 로드되는가?
- 기록 정보가 정확하게 표시되는가?
- 비로그인 사용자도 접근할 수 있는가?
```

**예상 결과:**
- 공유 페이지 정상 로드
- 기록 정보 정확히 표시
- 비로그인 사용자 접근 가능

---

### 시나리오 6-5: Open Graph 메타 태그 확인

**프롬프트:**
```
공유 페이지에서:

1. 개발자 도구를 엽니다
2. Elements 탭에서 `<head>` 섹션을 확인합니다
3. Open Graph 메타 태그가 있는지 확인합니다:
   - og:title
   - og:description
   - og:image
   - og:url
   - og:type
4. 각 메타 태그의 값이 정확한지 확인합니다
5. Facebook 공유 디버거를 사용하여 확인합니다 (선택사항)

확인 사항:
- Open Graph 메타 태그가 정상적으로 설정되어 있는가?
- 메타 태그 값이 정확한가?
- 이미지 URL이 정상적으로 작동하는가?
```

**예상 결과:**
- Open Graph 메타 태그 정상 설정
- 메타 태그 값 정확
- 이미지 URL 정상 작동

### 시나리오 6-7: 공개/비공개 설정

**프롬프트:**
```
기록 작성 또는 수정 페이지에서:

1. "공개" 또는 "비공개" 설정 옵션을 찾습니다
2. 공개로 설정합니다
3. 기록을 저장합니다
4. 공유 링크를 생성합니다
5. 공유 링크가 정상적으로 작동하는지 확인합니다
6. 비공개로 설정합니다
7. 기록을 저장합니다
8. 공유 링크가 작동하지 않는지 확인합니다

확인 사항:
- 공개/비공개 설정이 정상적으로 작동하는가?
- 공개 기록은 공유 링크로 접근 가능한가?
- 비공개 기록은 공유 링크로 접근 불가능한가?
```

**예상 결과:**
- 공개/비공개 설정 정상 작동
- 공개 기록 공유 링크 접근 가능
- 비공개 기록 공유 링크 접근 불가
