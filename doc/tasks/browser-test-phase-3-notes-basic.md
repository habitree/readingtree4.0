# Phase 3: 기록 관리 기능 테스트 (기본)

**단계:** 3/9  
**예상 소요 시간:** 15-20분  
**필수 선행 조건:** Phase 2 완료 (책 추가 완료)

---

## 🎯 테스트 목표

1. 텍스트 기록 작성이 정상 작동하는지 확인
2. 기록 목록 조회가 정상 작동하는지 확인
3. 기록 상세 조회가 정상 작동하는지 확인
4. 기록 수정이 정상 작동하는지 확인
5. 기록 삭제가 정상 작동하는지 확인

---

## 📋 테스트 시나리오

### 시나리오 3-1: 텍스트 기록 작성 (필사)

**프롬프트:**
```
책 상세 페이지에서:

1. "기록 작성" 버튼을 클릭합니다
2. 기록 작성 페이지(/notes/new?bookId=[bookId])로 이동하는지 확인합니다
3. 기록 유형을 "필사"로 선택합니다
4. 기록 내용을 입력합니다 (예: 인상 깊었던 문장)
5. 페이지 번호를 입력합니다 (선택사항)
6. 태그를 추가합니다 (선택사항, 예: "인상깊음", "명언")
7. "저장" 버튼을 클릭합니다
8. 기록이 성공적으로 저장되었는지 확인합니다 (토스트 메시지 또는 리다이렉트)
9. 개발자 도구 콘솔에서 에러가 없는지 확인합니다
10. 네트워크 탭에서 기록 생성 API 요청이 성공적으로 완료되었는지 확인합니다

확인 사항:
- 기록 작성 폼이 정상적으로 표시되는가?
- 입력 필드가 정상적으로 작동하는가?
- 유효성 검사가 작동하는가? (필수 필드)
- 기록이 성공적으로 저장되는가?
- 저장 후 적절한 페이지로 리다이렉트되는가?
```

**예상 결과:**
- 기록 작성 성공 토스트 메시지
- 기록 상세 페이지로 리다이렉트
- 입력한 내용이 정확하게 저장됨

---

### 시나리오 3-2: 텍스트 기록 작성 (메모)

**프롬프트:**
```
기록 작성 페이지에서:

1. 기록 유형을 "메모"로 선택합니다
2. 메모 내용을 입력합니다
3. 페이지 번호를 입력합니다 (선택사항)
4. 태그를 추가합니다 (선택사항)
5. "저장" 버튼을 클릭합니다
6. 기록이 성공적으로 저장되었는지 확인합니다
7. 개발자 도구 콘솔에서 에러가 없는지 확인합니다

확인 사항:
- 메모 유형 선택이 정상적으로 작동하는가?
- 메모 내용이 정확하게 저장되는가?
```

**예상 결과:**
- 메모 기록 성공적으로 저장
- 기록 상세 페이지로 리다이렉트

---

### 시나리오 3-3: 기록 목록 조회

**프롬프트:**
```
기록 목록 페이지(/notes)로 이동합니다:

1. 페이지가 정상적으로 로드되는지 확인합니다
2. 작성한 기록들이 목록에 표시되는지 확인합니다
3. 기록 정보(유형, 내용 미리보기, 책 제목, 날짜)가 정상적으로 표시되는지 확인합니다
4. 기록 유형 배지가 정상적으로 표시되는지 확인합니다
5. 개발자 도구 콘솔에서 에러가 없는지 확인합니다
6. 네트워크 탭에서 기록 목록 조회 API 요청이 성공적으로 완료되었는지 확인합니다

확인 사항:
- 기록 목록이 정상적으로 표시되는가?
- 기록 정보가 정확하게 표시되는가?
- 기록이 없을 때 적절한 메시지가 표시되는가?
- 페이지네이션이 정상적으로 작동하는가? (기록이 많을 경우)
```

**예상 결과:**
- 기록 목록 정상 표시
- 기록 정보 정확히 표시
- 기록이 없을 때 "기록 작성하기" 안내 표시

---

### 시나리오 3-4: 기록 상세 조회

**프롬프트:**
```
기록 목록에서:

1. 기록 목록 중 하나를 클릭합니다
2. 기록 상세 페이지(/notes/[id])로 이동하는지 확인합니다
3. 기록 상세 정보(유형, 내용, 책 정보, 페이지 번호, 태그, 작성일)가 정상적으로 표시되는지 확인합니다
4. "수정" 버튼이 표시되는지 확인합니다
5. "삭제" 버튼이 표시되는지 확인합니다
6. 개발자 도구 콘솔에서 에러가 없는지 확인합니다
7. 네트워크 탭에서 기록 상세 조회 API 요청이 성공적으로 완료되었는지 확인합니다

확인 사항:
- 기록 상세 정보가 정확하게 표시되는가?
- 기록 내용이 정확하게 표시되는가?
- 태그가 정상적으로 표시되는가?
- 수정/삭제 버튼이 정상적으로 표시되는가?
```

**예상 결과:**
- 기록 상세 정보 정상 표시
- 기록 내용 정확히 표시
- 태그 정상 표시
- 수정/삭제 버튼 표시

---

### 시나리오 3-5: 기록 수정

**프롬프트:**
```
기록 상세 페이지에서:

1. "수정" 버튼을 클릭합니다
2. 기록 수정 페이지(/notes/[id]/edit)로 이동하는지 확인합니다
3. 기존 기록 내용이 폼에 미리 채워져 있는지 확인합니다
4. 기록 내용을 수정합니다
5. 태그를 추가/수정합니다
6. "저장" 버튼을 클릭합니다
7. 기록이 성공적으로 수정되었는지 확인합니다 (토스트 메시지 또는 리다이렉트)
8. 수정된 내용이 기록 상세 페이지에 반영되었는지 확인합니다
9. 개발자 도구 콘솔에서 에러가 없는지 확인합니다
10. 네트워크 탭에서 기록 수정 API 요청이 성공적으로 완료되었는지 확인합니다

확인 사항:
- 기록 수정 폼이 정상적으로 표시되는가?
- 기존 내용이 정확하게 미리 채워져 있는가?
- 수정이 성공적으로 저장되는가?
- 수정된 내용이 즉시 반영되는가?
```

**예상 결과:**
- 기록 수정 성공 토스트 메시지
- 기록 상세 페이지로 리다이렉트
- 수정된 내용이 정확하게 반영됨

---

### 시나리오 3-6: 기록 삭제

**프롬프트:**
```
기록 상세 페이지에서:

1. "삭제" 버튼을 클릭합니다
2. 삭제 확인 다이얼로그가 표시되는지 확인합니다
3. 삭제를 확인합니다
4. 기록이 성공적으로 삭제되었는지 확인합니다 (토스트 메시지 또는 리다이렉트)
5. 기록 목록에서 삭제된 기록이 사라졌는지 확인합니다
6. 개발자 도구 콘솔에서 에러가 없는지 확인합니다
7. 네트워크 탭에서 기록 삭제 API 요청이 성공적으로 완료되었는지 확인합니다

확인 사항:
- 삭제 확인 다이얼로그가 정상적으로 표시되는가?
- 삭제가 성공적으로 완료되는가?
- 삭제 후 적절한 페이지로 리다이렉트되는가?
- 삭제된 기록이 목록에서 사라지는가?
```

**예상 결과:**
- 삭제 확인 다이얼로그 표시
- 삭제 성공 토스트 메시지
- 기록 목록 또는 책 상세 페이지로 리다이렉트
- 삭제된 기록이 목록에서 제거됨

---

### 시나리오 3-7: 권한 확인 (선택사항)

**프롬프트:**
```
다른 사용자의 기록에 접근하려고 시도합니다:

1. 공개된 기록의 공유 링크를 통해 접근해봅니다 (/share/notes/[id])
2. 다른 사용자의 기록 상세 페이지에 직접 접근하려고 시도합니다
3. 수정/삭제 버튼이 표시되지 않는지 확인합니다
4. 개발자 도구 콘솔에서 에러가 없는지 확인합니다

확인 사항:
- 다른 사용자의 기록을 조회할 수 있는가? (공개 기록인 경우)
- 다른 사용자의 기록을 수정/삭제할 수 없는가?
- 권한 오류가 적절하게 처리되는가?
```

**예상 결과:**
- 공개 기록은 조회 가능
- 다른 사용자의 기록은 수정/삭제 불가
- 권한 오류 시 적절한 에러 메시지 표시
